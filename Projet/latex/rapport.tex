\section{Introduction}

Le but de ce projet est de classifier des textes afin de pouvoir nous former aux divers outils de traitement du langage naturel. Pour cela, nous avons deux jeux de données différents : un jeu de données contenant des critiques de films, et un autre contenant des discours de Jaques Chirac et de François Mitterrand.\\
Nous avons divisé notre travail en plusieurs parties. Dans un premier temps, nous avons fait de l'analyse des données et du pré traitement des données. Ensuite nous avons pu commencer le travail de classification. Nous avons exploré plusieurs techniques de classification vu lors des TME et du cours.

Cependant, après le prétraitement des données, les courbes de Zipf deviennent moins linéaires, ce qui indique que les mots les plus fréquents ont été supprimés et que les mots restants sont plus équilibrés en termes de fréquence.\\

Ce prétraitement des données est le premier que nous avons implémenté. Il ne donnait pas de résultats satisfaisants donc nous nous sommes dirigés vers la TF-IDF pour faire du prétraitement des données.\\

La TF-IDF (Term Frequency – Inverse Document Frequency) consiste à attribuer un poids à chaque mot en fonction de son importance dans un document et dans l’ensemble du corpus. La composante TF (Term Frequency) mesure la fréquence d’un mot dans un document, tandis que la composante IDF (Inverse Document Frequency) pénalise les mots apparaissant fréquemment dans de nombreux documents. Ainsi, un mot fréquent dans un document mais rare dans le corpus aura un poids élevé, ce qui le rend plus discriminant pour la classification.\\

Nous avons utilisé la TF-IDF de la librairie Scikit-learn, beaucoup de configurations ont été testées et nous nous sommes arrêtés sur celle qui donnait les meilleurs résultats. 

Pour le jeu de données des films, nous avons utilisé les paramètres \texttt{max\_df = 1}, \texttt{min\_df = 3} et \texttt{sublinear\_tf = True}. Le paramètre \texttt{min\_df = 3} permet de supprimer les mots apparaissant dans moins de trois documents, ce qui réduit le bruit lié aux mots très rares ou aux erreurs typographiques. Le paramètre \texttt{max\_df = 1} signifie que nous ne filtrons pas explicitement les mots très fréquents, car la pondération TF-IDF se charge déjà de diminuer leur importance.\\

L'option \texttt{sublinear\_tf = True} applique une transformation logarithmique sur la fréquence des mots, donnée par \(1 + \log(\text{tf})\), ce qui permet de limiter l'influence des mots très fréquents dans un document et d'obtenir une représentation plus équilibrée.\\

Enfin, nous avons choisi de désactiver le prétraitement manuel (mise en minuscules, suppression des accents, etc.), car l'implémentation de Scikit-learn intègre déjà certaines étapes de normalisation, notamment la conversion en minuscules, et les résultats expérimentaux ont montré que ce prétraitement supplémentaire n'apportait pas d'amélioration significative des performances.\\



\section{Classification}

Dans cette section, nous présentons les différentes méthodes de classification testées sur nos jeux de données. Nous comparons leurs performances afin d’identifier les approches les plus efficaces pour chaque tâche.




\subsection{Méthodologie}

Nous utilisons une représentation TF-IDF avec des n-grammes (1,2). Les modèles sont évalués à l’aide d’une validation croisée (K-Fold) afin d’obtenir des résultats robustes.

Pour le jeu de données des films, nous utilisons une validation croisée à 5 plis, tandis que pour les discours politiques nous utilisons une validation croisée à 10 plis.


Les métriques utilisées sont :
\begin{itemize}
\item Accuracy
\item F1-score
\item Précision
\item Rappel
\item ROC-AUC (pour les discours)
\end{itemize}

\subsection{Modèles testés}

Nous avons expérimenté plusieurs modèles de classification :

\begin{itemize}
\item SVM linéaire (\texttt{LinearSVC})
\item Régression logistique
\item Naive Bayes multinomial
\item Transformers (BERT)
\end{itemize}


\subsubsection{Transformers}

Le transformer est la méthode la plus avancée que nous avons testée mais aussi celle que nous avons le moins apréciée en raison de sa complexité et de son coût en calcul. Pour le dataset de films le modèle qui a été choisi est le roberta-base. Ce modèle est fait pour des taches de classification de texte. Notre tache a été de fine-tuner le modèle, pour cela nous avons utilisé un tokenizer avec une taille de 512 tokens, un learning rate de $1.5e-5$ un batch size de 32. Les transformers sont adaptées pour les tâches avec de grandes données équilibrées et qui comportent des données corrélées. Cela explique les bons résultats que nous avons obtenus sur le dataset de films.\\

Contrairement aux analyses de films nous avons utilisé camembert-base comme modèle qui est adapté pour le français. Nous avons utilisé un tokenizer avec une longueur maximale de 128 tokens car les discours sont juste des phrases donc plus courtes que les films. Nous avons choisi adamW comme optimiseur avec un learning rate de $2e-5$. Le point le plus important est la Cross Entropy avec poids dynamique pour compenser le déséquilibre des classes et donc possiblement compenser le biais. De plus la classification de présidents est plus complexe que celle des films car la différence est dans le style de parole avec souvent un vocabulaire commun.\\

Pour les deux transformers nous avons essayé plusieurs learning rate, nombre d'epochs et aussi ajouté du dropout. Nous avons aussi essayé d'undersampler le dataset de discours de présidents.\\

En conclusion le transformer a donné de très bons résultats sur le detaset de films mais les résultats sur les discours de présidents sont décevants. Nous n'avons pas réussi à gérer le déséquilibre des classes et le modèle a tendance à prédire la classe majoritaire (Chirac) pour la grande majorité des phrases. Une amélioration possible serait de faire de la data augmentation pour le jeu de données des discours de présidents.\\
